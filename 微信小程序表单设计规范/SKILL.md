# 微信小程序表单设计规范

## description
微信小程序表单设计规范技能。提供微信小程序中表单组件的最佳实践和标准规范，包括表单布局、输入控件、错误提示、表单验证等设计指导。当用户提到"小程序表单"、"表单设计"、"输入框"、"表单验证"、"表单报错"时使用此 Skill。

## guidelines

### 表单设计基本原则
1. **减少输入**：尽量减少用户手动输入，利用接口和选择控件
2. **清晰标注**：明确标注必填项和输入要求
3. **及时反馈**：输入过程中提供实时反馈
4. **错误明确**：出错时明确指出错误位置和原因

### 输入控件规范
| 控件类型 | 使用场景 | 设计要点 |
|---------|---------|---------|
| 文本输入框 | 短文本输入 | 提供占位提示，限制输入长度 |
| 多行文本框 | 长文本输入 | 显示字数统计，支持自动高度 |
| 数字输入框 | 数值输入 | 提供步进器或滑块辅助 |
| 选择器 | 选项选择 | 使用picker组件，减少键盘输入 |
| 开关 | 二选一设置 | 使用switch组件，状态明确 |
| 多选框 | 多选项选择 | 使用checkbox组件，显示选中状态 |
| 单选框 | 单选项选择 | 使用radio组件，互斥关系明确 |

### 表单布局规范
- 表单项垂直排列，便于逐项填写
- 标签与输入框对齐（建议标签宽度一致）
- 相关表单项分组显示
- 必填项使用(*)或文字标注
- 提供表单填写说明或示例

### 表单错误提示
根据不同场景选择合适的报错类型：

1. **原位报错**
   - 在出错表单项下方显示错误提示
   - 使用红色文字(#E64340)
   - 同时标识出错的输入框
   - 适用于单个字段即时校验

2. **公告报错**
   - 在表单顶部显示错误汇总
   - 列出所有错误项及原因
   - 适用于提交时批量校验

3. **模态报错**
   - 使用模态对话框显示严重错误
   - 需用户确认后才能继续
   - 适用于阻断性错误提示

### 表单验证时机
- **即时验证**：输入完成后立即验证，适合格式校验
- **失焦验证**：输入框失去焦点时验证，适合内容校验
- **提交验证**：提交时统一验证，适合关联性校验

### 表单交互优化
- 利用摄像头识别接口（如银行卡识别）
- 利用地理位置接口
- 提供搜索历史快捷选项
- 使用选择器代替键盘输入
- 自动填充已知信息

### 设计注意事项
- 表单项较多时分组显示，避免一屏过多
- 提供明确的提交按钮和取消选项
- 提交后显示加载状态
- 成功后给予明确反馈
- 异常状态提供解决方案，有路可退
