# 微信小程序色板设计规范

## guidelines

### 文字用色规范
微信小程序文字颜色分为以下层级：

| 颜色类型 | 色值 | 使用场景 |
|---------|------|---------|
| 主要文字 | #000000 / #333333 | 重要标题、主要内容 |
| 次要文字 | #666666 | 辅助说明、次要信息 |
| 提示文字 | #999999 | 提示语、禁用状态 |
| 链接文字 | #576B95 | 可点击链接 |
| 错误提示 | #E64340 | 错误信息、警告提示 |

### 彩色色板规范
微信小程序提供标准彩色色板：

| 颜色名称 | 色值 | 使用场景 |
|---------|------|---------|
| 微信绿（主色） | #07C160 | 品牌色、成功状态、主要操作 |
| 警告橙 | #FA9D3B | 警告提示、提醒信息 |
| 错误红 | #E64340 | 错误状态、删除操作 |
| 链接蓝 | #576B95 | 链接文字、次要操作 |

### 颜色使用原则
1. **品牌一致性**：使用微信绿色作为品牌主色，保持与微信生态的一致性
2. **功能语义化**：不同颜色代表不同功能含义（成功=绿，警告=橙，错误=红）
3. **层级分明**：通过颜色深浅区分信息层级
4. **可访问性**：确保文字与背景色对比度足够，满足可读性要求

### 配色建议
- 主色调建议使用微信绿色(#07C160)，体现微信品牌特征
- 页面背景建议使用浅色(#F7F7F7)或白色(#FFFFFF)
- 重要操作按钮使用主色或高对比度颜色
- 禁用状态使用灰色(#CCCCCC)表示

### 小程序菜单配色
小程序菜单提供深浅两种配色方案：
- **深色方案**：适用于浅色背景页面
- **浅色方案**：适用于深色背景页面

开发者可根据页面设计风格选择合适的菜单配色，确保菜单与页面整体风格协调。

### 设计注意事项
- 避免使用过多颜色，保持页面简洁
- 同类功能使用相同颜色，保持一致性
- 考虑色盲用户的可访问性
- 深色模式下需调整颜色方案
